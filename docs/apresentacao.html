Problema e Solução

"Como unificar o trabalho do instrutor no desktop com o acompanhamento do aluno pelo celular?"

Problemas Identificados

  • Instrutores usam planilhas ou papel para fichas
  • Alunos perdem a ficha ou não registram cargas
  • Sem feedback do aluno para o instrutor
  • Histórico de evolução indisponível no treino

Solução

  • Desktop: JavaFX para instrutor (cadastro, fichas, dashboards)
  • Mobile: SPA para aluno (treino, progresso, feedback)
  • Um único banco PostgreSQL compartilhado

Diferenciais Técnicos

GIPHY

  • Busca automática por nome + grupo muscular
  • Em lote com threads paralelas (AtomicInteger)
  • Auto-preenchimento na API mobile
  • Chave via env / .properties / sistema

Frontend — SPA

  • HTML, CSS e JS puros — sem frameworks externos
  • 4 abas: Feed, Treino, Performance, Perfil
  • Gráfico semanal em Canvas 2D
  • Armazenamento local (localStorage) e API centralizada

Concorrência

  • CachedThreadPool no servidor HTTP
  • EntityManager-per-request (thread-safe via isolamento)
  • Platform.runLater() para UI JavaFX

Padrões de Projeto

  • DAO, Singleton, MVC

Desafios Técnicos

1. Integrar servidor HTTP com o ciclo de vida do JavaFX

Servidor inicia via new Thread(() -> ServidorMobile.iniciar()).start() no initialize() do PainelPrincipal.

2. Thread-safety com EntityManager não thread-safe

Cada handler HTTP cria seu próprio EntityManager — padrão EntityManager-per-request.

3. Consistência transacional entre EntityManagers (melhoria futura)

Operações concorrentes em EntityManagers diferentes podem gerar inconsistência. Ideal seria um Transaction Manager.
